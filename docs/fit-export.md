# FIT workout export (plans → watch)

Every planned session can be downloaded as a structured `.fit` workout file and loaded
onto a Garmin watch, giving on-wrist step-by-step guidance with pace alerts. This uses
only the **official** Garmin FIT SDK — no Garmin account or API involved.

## Using it

1. Open a workout (dialog **.FIT** button, or the button on the detail page).
2. A file like `runplan-2026-07-28-threshold.fit` downloads.
3. Connect the watch over USB and copy the file into its **`GARMIN/Workouts`** folder
   (on some models `GARMIN/NewFiles`). Note: Garmin Connect's *web import* only accepts
   activities, not workout files — USB copy is the reliable route.
4. On the watch: **Training → Workouts** → pick the session.

## What gets encoded

`src/lib/fit/steps.ts` translates the planned session into FIT workout steps:

| Session element | FIT encoding |
| --- | --- |
| Easy / long / recovery runs | one distance step at the session's pace band |
| Warm-up / cool-down segments | distance steps at the runner's easy zone |
| Interval reps (`5 × 1000m @ interval pace, 2:30 jog recovery`) | repeat block: work step + timed jog recovery × N |
| Time-based reps (`2 × 15 min @ threshold, 3 min jog`) | repeat block with timed work steps |
| Strides (`6 × 20s strides`) | main run step, then 20s @ rep pace / 60s recovery × N |
| Long run with MP finish (`final 12 km @ marathon pace`) | easy portion + marathon-pace block |
| Continuous tempo (`40 min continuous @ threshold`) | timed step at threshold |
| Structured work without explicit warm-up (taper sharpener) | bracketed with lap-press warm-up/cool-down steps |

Details:

- Pace targets are ranges (custom speed targets). Single-pace targets are widened by
  ~2% / ≥5 s/km so the watch alert zone is usable.
- Warm-up/cool-down/stride paces come from the runner's **current** VDOT zones
  (`paceZones(plan.currentVdot)`), not the goal zones.
- Segment labels are generated by the plan engine with a small fixed grammar; anything
  unparseable falls back to a single steady step, so the export never fails.
- Encoding uses `@garmin/fitsdk`'s `Encoder` (`src/lib/fit/encode.ts`). Raw wire scaling
  (metres ×100, seconds ×1000, m/s ×1000) applies to the generic `durationValue` /
  `targetValue` fields; repeats use `durationType: repeatUntilStepsCmplt`.

The export endpoint is `GET /api/workouts/<id>/fit` (session auth; rest days 400).
